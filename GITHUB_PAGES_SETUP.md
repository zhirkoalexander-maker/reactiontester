# GitHub Pages Setup - Инструкции по развертыванию

## ✅ Проект уже загружен на GitHub!

Репозиторий: https://github.com/zhirkoalexander-maker/reactiontester

## 🚀 Включение GitHub Pages

### Шаг 1: Перейти в настройки репозитория
1. Откройте ваш репозиторий на GitHub
2. Нажмите на вкладку **Settings** (Параметры)

### Шаг 2: Включить GitHub Pages
1. Найдите раздел **Pages** в левом меню
2. Выберите источник: **Deploy from a branch**
3. Выберите ветку: **main**
4. Выберите папку: **/(root)**
5. Нажмите **Save**

### Шаг 3: Ждите развертывания
GitHub обычно развертывает сайт в течение 1-2 минут.

## 🌐 Доступ к приложению

После включения GitHub Pages приложение будет доступно по адресу:

```
https://zhirkoalexander-maker.github.io/reactiontester/
```

## ✨ PWA Функции

### После развертывания на GitHub Pages:

1. **Установка как приложение:**
   - Откройте сайт в браузере
   - На Android: Меню → "Установить приложение"
   - На iOS: Поделиться → "На главный экран"

2. **Офлайн работа:**
   - Service Worker автоматически кэширует все ресурсы
   - Приложение работает без интернета

3. **Брендирование:**
   - Собственный иконка ⚡
   - Собственное имя "Тест Реакции"
   - Темный интерфейс

## 🔧 Технические детали

### Service Worker
- Регистрируется автоматически при загрузке страницы
- Кэширует CSS, HTML, JS и манифест
- Поддерживает стартегию "cache first" с fallback на сеть

### Манифест
- Устанавливает цвета темы: #0a0e27 (темный), #ff006e (розовый)
- Определяет иконки для разных размеров экранов
- Поддерживает maskable иконки для адаптивности

## 🐛 Решение проблем

### Service Worker не регистрируется?
- Убедитесь что используется HTTPS (GitHub Pages использует HTTPS автоматически)
- Проверьте консоль браузера (F12 → Console)
- Очистите кэш браузера и перезагрузите страницу

### Иконка не отображается?
- Используются SVG иконки встроенные в manifest
- Это работает во всех современных браузерах

### Приложение не устанавливается?
- Требуется HTTPS (GitHub Pages это предоставляет)
- Требуется маниле с явным указанием start_url
